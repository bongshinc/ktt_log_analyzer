# KT텔레캅 지문리더기 로그분석기

KT텔레캅 출입통제 시스템의 인증 로그를 분석하는 **오프라인 HTML 단일 파일** 도구입니다.
서버 설치 없이 브라우저에서 XLS 파일을 업로드하는 것만으로 즉시 동작합니다.

아래의 JS 파일을 이용합니다. 오프라인 사용을 위해 다운로드하여 HTML 파일과 같은 디렉터리에 두거나 없는 경우 인터넷에 연결하여 사용합니다.
| [SheetJS CE (xlsx)](https://sheetjs.com/) | 0.20.3 | Apache 2.0 | [다운로드](https://cdn.sheetjs.com/xlsx-0.20.3/package/dist/xlsx.full.min.js) |
| [Chart.js](https://www.chartjs.org/) | 4.4.0 | MIT | [다운로드](https://cdn.jsdelivr.net/npm/chart.js@4.4.0/dist/chart.umd.min.js) |

---

## 주요 기능

| 탭 | 내용 |
|---|---|
| 📅 **월별 현황** | 월별 인증 횟수(해제·경계·출입) 집계 + 리더기별 사용 현황 바 차트 |
| 👆 **지문(FP) 분석** | 지문 인증 기록만 분리해 월별 정렬·검색 |
| 💳 **카드(RF) 분석** | RF 카드 인증 기록만 분리해 월별 정렬·검색 |
| 📍 **리더기 통계** | 리더기별 사용자 인증 현황 차트 및 테이블 |
| ⚠️ **관리 대상 사용자** | 최근 90일 이내 출입 기록이 없는 등록 사용자 목록 |
| 🚫 **미등록 사용자** | 사용자 명부에 없는 인증번호의 출입 기록 + 위험도 분류 |

---

## 파일 구성

```
출입인증관리.html       ← 메인 애플리케이션 (이 파일 하나로 실행)
xlsx.full.min.js        ← SheetJS 오프라인용 (별도 다운로드, 아래 참조)
chart.umd.min.js        ← Chart.js 오프라인용 (별도 다운로드, 아래 참조)
```

> `xlsx.full.min.js` / `chart.umd.min.js` 가 HTML 파일과 **같은 폴더**에 있으면 완전 오프라인으로 동작합니다.
> 두 파일이 없어도 인터넷을 통해 CDN에서 자동으로 가져오므로 기본 사용에는 문제 없습니다.

## 오프라인 사용 준비 (선택)

인터넷이 차단된 환경에서 사용하려면 아래 두 파일을 `출입인증관리.html`과 **같은 폴더**에 저장하세요.

### 1. SheetJS (xlsx.full.min.js)

SheetJS Community Edition(CE)은 [Apache 2.0](https://www.apache.org/licenses/LICENSE-2.0) 오픈소스입니다.

```
다운로드 URL:
https://cdn.sheetjs.com/xlsx-0.20.3/package/dist/xlsx.full.min.js
```

브라우저 주소창에 위 URL을 입력한 뒤 **파일로 저장** (`Ctrl+S` / `Cmd+S`) 하여
`xlsx.full.min.js` 이름으로 저장하세요.

> ⚠️ sheetjs.com 에서 배포하는 최신 패키지는 상업용 라이선스가 적용될 수 있습니다.
> 반드시 위의 `cdn.sheetjs.com` CE 버전 링크를 사용하세요.

### 2. Chart.js (chart.umd.min.js)

Chart.js는 [MIT](https://github.com/chartjs/Chart.js/blob/master/LICENSE.md) 오픈소스입니다.

```
다운로드 URL:
https://cdn.jsdelivr.net/npm/chart.js@4.4.0/dist/chart.umd.min.js
```

브라우저 주소창에 위 URL을 입력한 뒤 **파일로 저장** (`Ctrl+S` / `Cmd+S`) 하여
`chart.umd.min.js` 이름으로 저장하세요.

## 사용 방법

1. `출입인증관리.html`을 브라우저로 엽니다.
2. **사용자 명부** 업로드 — 사용자 정보가 담긴 xls / xlsx 파일 (파일명 무관)
3. **출입 로그** 업로드 — 인증 기록 xls 파일 (파일명 무관, 여러 파일 동시 업로드 가능)
4. 각 탭에서 결과를 확인하고 필요 시 엑셀로 내보냅니다.

> 파일명은 자유롭게 사용해도 됩니다. 파일 내부의 **헤더 컬럼명**을 자동으로 인식하여 구분합니다.
> - 사용자 명부: `사원번호` 또는 `인증번호` 컬럼이 있는 파일
> - 출입 로그: `인증일시` 컬럼이 있는 파일

## 입력 파일 형식

### 사용자 명부

사용자 등록 정보 파일입니다. 첫 번째 시트의 헤더 행을 자동으로 감지합니다.
리더기나 관리프로그램 또는 수동으로 작성한 아래와 같은 표의 엑셀 파일입니다.

| 인증번호 | 사원번호 | 이름 | 전화번호 | 근무여부 | 직위 |
|:---:|:---:|:---:|:---:|:---:|:---:|
| 1234 | EMP001 | 홍길동 | 910-1234-5678 | 재직 | 팀장 |
| 5678 | EMP002 | 김철수 | 910-9876-5432 | 퇴직 | 대리 |
| 9012 | EMP003 | 이영희 | 910-5555-1234 | 휴직 | 사원 |
| 3456 | EMP004 | 박민준 | 910-2222-3333 | 재직 | 과장 |

- **인증번호** 또는 **사원번호** 중 하나를 키로 사용합니다.
- 열 이름에 해당 단어가 포함되면 자동 인식합니다 (예: `사원 번호`, `인증 ID` 등).

|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| 2026-01-06 08:32:14 | 화 | 1234 | EMP001 | 홍길동 | 1층사무실(주장치) | 해제 | 성공 | FP | 관리팀 | 팀장 | 평일 | 재직 |
| 2026-01-06 08:35:02 | 화 | 5678 | EMP002 | 김철수 | 1층후문 | 출입 | 성공 | RF | 기술팀 | 대리 | 평일 | 퇴직 |
| 2026-01-06 18:21:33 | 화 | 1234 | EMP001 | 홍길동 | 1층사무실(주장치) | 경계 | 성공 | FP | 관리팀 | 팀장 | 평일 | 재직 |
| 2026-01-07 08:30:05 | 수 | 3456 | EMP004 | 박민준 | B1 방송실 | 해제 | 성공 | FP | 기술팀 | 과장 | 평일 | 재직 |

#### 인증모드 설명

| 모드 | 발생 위치 | 의미 |
|:---:|:---:|---|
| **해제** | 주장치(보안 제어반) | 경계 해제 — 업무 시작 전 보안 시스템을 비활성화 |
| **경계** | 주장치(보안 제어반) | 경계 설정 — 퇴근·외출 시 보안 시스템을 활성화 |
| **출입** | 개별 도어 리더기 | 도어 통과 — 해제 상태에서 특정 도어를 개방 |

## 미등록 사용자 위험도 기준

사용자 명부에 없는 인증번호가 감지될 경우 아래 기준으로 위험도를 분류합니다.

| 등급 | 기준 | 조치 |
|:---:|---|---|
| 🔴 **UNAUTHORIZED** | **주장치(보안 제어반)**에서 해제·경계 인증이 감지된 미등록 번호. 보안 시스템을 직접 조작한 것으로 판단 | 즉각 보안 검토 필수 |
| 🟠 **WARNING** | **일반 도어 리더기(출입)**만 감지된 미등록 번호. 타인의 해제 후 동반 진입하거나 비경계 상태에서 통과한 것으로 추정 | 내부 확인 권장 |

---

## 관리 대상 사용자 기준

- 로그 파일의 **가장 최근 날짜**를 기준으로 **90일(약 1분기) 이내** 출입 기록이 없는 등록 사용자를 표시합니다.
- 퇴직·장기 휴직 등으로 인증 권한 회수 여부를 주기적으로 확인하기 위한 목록입니다.
